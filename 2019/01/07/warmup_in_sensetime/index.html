warmup in sensetime

2019-01-07 pytorch sensetime 评论

Warmup

​ 熟悉集群的使用，memcached缓存系统，多机多卡的训练方式以及公司内部使用的基于caffe的Sensenet的使用。参考网址算法平台

Pytorch

​ 一、前期准备：

  • 修改memcached_dataset.py文件：该文件主要是构建自己的dataset，补充的知识点为memcached系统，参考算法平台中相关的介绍Memcached缓存系统

  • 修改main.py文件：包含了命令行参数处理，数据处理，训练函数和验证函数

    1. main():利用argparse模块对命令行参数进行处理(参数设置)；根据命令行参数以及config.yaml文件中对于迭代次数，学习率等设置构建model，分别构建train和validate的dataset和dataloader。
    2. train():根据传入的train和validate的dataloader进行训练，主要补充的知识点为tensorboard logger的使用，并且记录top-1和top-5的train_accuracy。top-1是指imagenet训练出来的结果(softmax得到的概率值)最高者对应的种类和label一致时为预测正确，top-5是指训练出来的前5高的概率值对应的5个种类中有一个为label则认为预测正确。
    3. validate():和train流程一样，但记录的是validate_accuracy

  • 运行train.sh:文件中写明了srun时对应的各种参数，python运行时读取的配置文件的目录等参数。

    Sensenet

    ​ 一、对于caffe部分而言主要是了解各个prototxt配置文件的意义基础教程和区别

    • Imagenet_solver.prototxt （包含全局参数的配置的文件）
    • Imagenet_trainval.prototxt （包含训练和验证网络的配置的文件）
    • Imagenet_deploy.prototxt （包含测试网络的配置的文件）

    ​ 二、内存优化:参考地址

    ​ 在优化内存占用的过程中，主要使用了动态分配和释放内存的策略，为了减少 GPU 内存分配和释放的开销，SenseNet 提供了 GPU 内存池功能.

    • liveness考虑feature map的data/diff生存期，若其data/diff在后面不会再使用则将其释放。
    • Checkpoint 策略会在 Forward 过程中只保存少数几个 feature map（称为 Checkpoint 点），并释放其他的 feature map，这样将整个模型分成若干段，每段以一个 Checkpoint 点开始，到下一个 Checkpoint 点之前结束。在 Backward 时，每次处理一段的模型，从该段开始的 Checkpoint 点重新执行一遍 Forward 过程，得到该段的所有 feature map，然后执行通常的 Backward 过程。一段处理完成后，释放所有不需要的 feature map，然后开始执行下一段的 Backward。

    ​ 三、数据读取优化:参考地址

    ​ 使用了MultiDB存储方案而不是memcached，基本原理是将多个小文件打包成大文件，每次随机读取多个大文件到内存中，然后每次迭代取内存中的一个batch的数据。
